After User Makes an Edit

Steps (always inside git-integration branch):

Admin server updates project files (HTML, CSS, JS).

Stage all changes:

git add .


Commit changes with descriptive message:

git commit -m "AI Edit: <short description of edit>"


Create a version tag:

git tag vX   # Example: v1, v2, v3...


Push commit and tags to GitHub:

git push origin git-integration --tags


Do NOTs:

❌ Don’t push to main (test phase is only git-integration)

❌ Don’t reuse tags

❌ Don’t skip commits

3️⃣ Rollback to a Previous Version

Steps (still on git-integration branch):

List available tags:

git tag --list


Checkout git-integration branch:

git checkout git-integration


Reset branch to desired version tag:

git reset --hard v1


Push reset to GitHub (so remote reflects rollback):

git push origin git-integration --force


Notes:

✅ Use tags like v1, v2 instead of commit hashes

✅ After rollback, keep editing → commit → next tag (e.g., v4)

❌ Never rollback on main during test phase

4️⃣ Versioning Rules

Every edit = one commit + one tag

Tags must be sequential (v1, v2, v3…)

Rollback resets branch to chosen tag

After rollback, next edit continues as new tag (v4)

5️⃣ User Interaction in Admin UI

Version History Dropdown

Shows available tags (v1, v2, v3) with timestamp + description

Preview (Optional)

git show v1:index.html → render in preview panel (read-only)

Rollback

User clicks rollback → admin server executes:

git checkout git-integration
git reset --hard v1
git push origin git-integration --force


After Rollback

User edits again → new commit + tag (v4)
